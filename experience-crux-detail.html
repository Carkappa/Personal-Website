🎯 The Problem

Climbing coaches have always faced a fundamental challenge: how to provide actionable, data-driven feedback to athletes in real-time. Traditional methods rely on coaches' visual inspection—fast but inconsistent and prone to human bias.

What if coaches could have an AI assistant that could:

  • Analyze every frame of a climbing session automatically
  • Track body positioning and movement patterns with precision
  • Measure movement efficiency quantitatively
  • Generate structured coaching reports in seconds, not hours

That was the vision behind BoulderingAI—a premium video analysis platform built to transform climbing coaching through computer vision and deep learning.

YOLOv11 detects climbing holds (crimps, jugs, slopers, pockets) with 95%+ accuracy across diverse boulder problems and lighting conditions

The Solution: A Complete AI Pipeline

I engineered a full-stack climbing analytics platform combining computer vision, pose estimation, FastAPI backend, and modern UI—all designed to deliver actionable insights from raw climbing videos.

Technical Architecture

The platform is built on a modern full-stack foundation:

  • Backend — FastAPI for async job processing and REST API
  • Frontend — Dark theme UI with real-time sparkline charts, frame previews, and confetti animations
  • Processing — OpenCV for video frame extraction, NumPy for feature computation
  • Data Persistence — Structured JSON storage for metrics, reports, and session history
  • DevOps — Docker containerization and auto-commit versioning for reproducibility

Key Accomplishments

  • 🎯 High Accuracy Models — Achieved 95%+ precision on hold detection and keypoint localization across varied climbing angles and lighting conditions
  • 📊 6 Real-Time Metrics — Built and animated 6 concurrent sparkline charts tracking performance across static stability, movement fluidity, and balance
  • ⚡ 30fps Throughput — Optimized pipeline processes full climbing sessions at video framerate with minimal latency
  • 🎨 Premium UX — Developed interactive dashboard with frame previews, confetti celebrations, toast notifications, and downloadable reports
  • 🔄 Automated DevOps — Implemented auto-commit system for tracking all development changes and ensuring reproducibility

Key Learnings

Working on BoulderingAI taught me several critical lessons:

  • Data Quality > Model Complexity — Spending extra time on careful labeling in Roboflow paid massive dividends, reducing model errors far more than hyperparameter tuning
  • Domain Knowledge Matters — Understanding climbing biomechanics helped me identify which metrics actually matter to coaches vs. purely technical measures
  • Deployment is Half the Work — Building models is 20%; the real challenge is creating a reliable, fast pipeline that coaches can use daily
  • Real-time Feedback is Hard — Balancing model accuracy with inference speed required creative optimization and careful feature selection
  • Users Know What They Want — Iterating with actual climbers and coaches revealed that dashboards matter as much as algorithms

    2. Model Training: YOLO + Pose Estimation

    I fine-tuned two models working in tandem:

    • YOLOv11 (Object Detection) — Detects climbing holds with bounding boxes, achieving high precision on varied boulder problems across angles and lighting
    • Pose Estimation — Tracks 17 keypoints per frame to extract body positioning, reach, and balance patterns

    Both models run on every frame, enabling frame-level analysis at 30fps video throughput.

    3. Movement Metrics & Post-Processing

    Raw detection outputs become meaningful insights through automated feature engineering:

    • Static Stability — Measures posture consistency across frames (shoulder/hip alignment variance)
    • Dynamic Movement — Tracks transition efficiency and fluidity between holds
    • Balance & Weight Distribution — Computes center-of-mass shifts relative to hold positions
    • Extension Quality — Measures reach patterns and positioning relative to target holds
    • Move Efficiency — Quantifies smoothness vs. jerky movements (velocity/acceleration analysis)
    • Duration Distribution — Tracks time spent on each hold and segment
